# Anim8orX v0.6.4

Real `.an8` import and large-file viewport correction.

## Fixed

- Object mode now behaves closer to Anim8or: it shows the active top-level object instead of drawing every object definition on top of itself.
- Scene mode can still show the whole loaded document.
- The hierarchy now lists top-level objects first, then the visible mesh nodes for the active object/scene.
- Imported mesh `base` orientation quaternions are applied, so rotated parts are no longer placed as if only their origin mattered.
- `subdivision { ... }` chunks with point/face data now load as renderable control meshes instead of disappearing.
- Large imports use an adaptive face budget so opening and orbiting big `.an8` files no longer spends seconds drawing every polygon in the GDI shell.

## Added

- Top-level `.an8` texture declarations are parsed and resolved beside the `.an8` file, including `Textures` and `Bump` folders when old absolute paths no longer exist.
- Object material definitions, diffuse RGB values, material map names, mesh material lists, UV arrays, and per-face material slots are parsed.
- Filled viewport rendering now uses cached material colors instead of painting every mesh the same default blue.
- The Material inspector shows selected mesh material slots and resolved texture-map counts.
