# Anim8orX v0.5.0

Viewport and branding pass.

## Added

- Added the provided `wlogo.png` asset to the repository and release package.
- Loaded the logo with GDI+ at runtime.
- Displayed the logo in the Anim8orX command strip.
- Used the logo as the window icon when available.
- Added `File > Open .an8...` using the Windows file picker.
- Added drag-and-drop loading for `.an8` files.
- Added `File > New` to reset to a default scene.
- Window title now reflects the loaded file.
- Replaced the flat screen-space viewport grid with a world-space 3D grid.
- Added perspective floor/depth/height grid guides.
- Added view-aware grid planes for Front, Back, Left, Right, Top, Bottom, and Ortho views.
- Added a custom dark in-viewport view selector from the orange viewport label:
  - All
  - Front
  - Back
  - Left
  - Right
  - Top
  - Bottom
  - Ortho
  - Perspective
- Wired native View menu commands to the same camera/view switching system.

## Fixed

- Selecting viewport views now updates the camera instead of being a no-op.
- The app no longer relies only on the white native menu dropdown for viewport switching.
- `.an8` loading is no longer limited to command-line launch or bundled sample files.

## Verification

- Built with MSVC/CMake Release configuration.
- Ran `Anim8orX.exe --smoke-test`.
- Extracted release zip and verified the GUI window remains open.
