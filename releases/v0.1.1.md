# Anim8orX v0.1.1

Startup reliability patch.

## Fixed

- Double-clicking `Anim8orX.exe` no longer appears to crash or close instantly.
- Launching without a file argument now loads the bundled `examples/cube.an8` sample.
- No-argument launch waits for Enter before closing so the test output remains visible.
- Updated stale usage text from `anim8orx_sandbox` to `Anim8orX`.

## Verification

- Built with MSVC/CMake Release configuration.
- Ran explicit file path: `Anim8orX.exe .\examples\cube.an8`.
- Ran no-argument startup path with bundled sample lookup.
- Extracted release zip and verified `Anim8orX.exe` loads the packaged cube sample.
